You are working on the CollabOS Command Center / Rose OS collaboration app.

Build a complete External Message Intake + High-Tech Collaboration Intelligence expansion for CollabOS, Rose/Carmen Mind Meld, and Company Brain review workflows.

Product names:
- Rose OS = full company operating system and AI assistant experience
- Command Center = work and project dashboard inside Rose OS
- Rose Brain = AI knowledge assistant and memory feature
- Company Brain = approved company records and institutional memory
- CollabOS Command Center = collaboration intelligence layer
- Mind Meld = private Rose/Carmen collaboration and alignment space

Core principle:
Make scattered thoughts, messages, ideas, to-dos, build requests, and decisions feel instantly organized, useful, and fun. Do not auto-approve anything. AI/rule-based outputs are recommendations until Rose, Carmen, or both approve them.

Do not auto-create approved tasks, decisions, SOPs, CRM changes, automations, or Company Brain records from messages. Everything captured from WhatsApp, Zoho Cliq, or manual entry must enter as a draft/review item first.

Build this as a safe intake + review + routing workflow.

SECTION 3: Smart Intake Classifier

Add a prominent “What Is This?” classifier for every intake item.

It should classify items as:
- Idea
- To-do
- Blocker
- Question
- Decision candidate
- Build request
- CRM / Zoho request
- Automation request
- Sensitive/private item
- Company Brain update suggestion
- Duplicate or related work
- No action / noise

Show a short plain-English explanation of why the classification was chosen.

Initial implementation can be rule-based if no live AI service exists.

Classification examples:
- Messages containing “Rose and Carmen,” “mind meld,” “Carmenfy,” “Rosify,” “strategy,” or “founder review” should suggest Rose/Carmen Mind Meld.
- Messages containing “Zoho,” “CRM,” “Cliq,” “automation,” “Deluge,” “Flow,” “form,” “field,” or “workflow” should route to Carmen review.
- Messages containing “decision,” “approved,” “final,” “launch,” “pricing,” “proposal,” or “client-facing” should route to Rose review or Rose and Carmen review depending on systems impact.
- Messages containing “todo,” “task,” “follow up,” “remind,” or “need to do” should suggest Command Center Task.
- Messages containing “idea,” “what if,” “we should,” or “new product” should suggest Idea Backlog or CollabOS Review Queue.
- Sensitive messages should not be routed into general Company Brain automatically.

SECTION 4: Magic Routing Buttons

Add intuitive action buttons to each item:

- Carmenfy This
- Rosify This
- Send to Mind Meld
- Turn Into To-Do
- Make It a Build Request
- Check for Duplicates
- Preserve in Company Brain
- Mark Sensitive / Private
- Archive / No Action

Button behavior:
- Carmenfy This routes systems, process, Zoho, automation, AI, and build-readiness items to Carmen review.
- Rosify This routes company direction, client-facing, pricing, launch, public positioning, final-decision, and sensitive direction items to Rose review.
- Send to Mind Meld routes private Rose/Carmen alignment items to Rose/Carmen Mind Meld.
- Turn Into To-Do creates a draft task.
- Make It a Build Request creates a draft build request.
- Preserve in Company Brain creates a proposed Company Brain update only, not an approved record.
- Check for Duplicates runs duplicate radar.
- Archive / No Action closes the item without creating work.
