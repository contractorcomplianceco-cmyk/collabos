You are working on the CollabOS Command Center / Rose OS collaboration app.

Build a new capability called External Message Intake for CollabOS + Mind Meld.

Purpose:
Allow Rose OS / CollabOS to receive relevant collaboration messages from Zoho Cliq and WhatsApp, classify them intelligently, and route them into the correct review area: Rose/Carmen Mind Meld, CollabOS Review Queue, Command Center tasks, idea backlog, build requests, or proposed Company Brain updates.

Important product names:
- Rose OS = full company operating system and AI assistant experience
- Command Center = work and project dashboard inside Rose OS
- Rose Brain = AI knowledge assistant and memory feature
- Company Brain = approved company records and institutional memory
- CollabOS Command Center = collaboration intelligence layer
- Mind Meld = private Rose/Carmen collaboration and alignment space

Core rule:
Do not auto-create approved tasks, decisions, SOPs, CRM changes, or Company Brain records from messages. Everything captured from WhatsApp or Zoho Cliq must enter as a draft/review item first.

Build this as a safe intake + review workflow.

Required Feature Set:

1. Add A New Section
Add a new app section named:
External Intake

Navigation label options:
- External Intake
- Message Intake
- Intake Queue

This section should show messages captured from:
- Zoho Cliq
- WhatsApp
- Manual test entry / sample message

2. Intake Sources

Phase 1 should support:
- Manual sample intake form for testing
- Zoho Cliq-ready webhook endpoint or mocked endpoint
- WhatsApp-ready webhook endpoint or mocked endpoint

If real credentials are not configured, clearly show:
Integration not connected
or
Demo / test mode

Do not imply live WhatsApp or live Zoho Cliq integration exists unless environment variables and server endpoints are actually configured.

3. Message Intake Schema

Each intake item should include:

- id
- source: zoho_cliq | whatsapp | manual
- sourceChannel
- senderName
- senderHandle
- senderRole if known
- receivedAt
- rawMessage
- cleanedSummary
- detectedType:
  - idea
  - todo
  - build_request
  - decision_candidate
  - blocker
  - question
  - process_update
  - CRM_or_Zoho_request
  - Rose_Carmen_mind_meld
  - company_brain_update_suggestion
  - ignore_or_noise
- suggestedDestination:
  - Rose/Carmen Mind Meld
  - CollabOS Review Queue
  - Command Center Task
  - Idea Backlog
  - Build Registry Suggestion
  - Requirements Registry Suggestion
  - Automation Registry Suggestion
  - Decision Log Suggestion
  - Company Brain Update Suggestion
  - No Action
- sensitivity:
  - normal
  - private_leadership
  - client_sensitive
  - HR_sensitive
  - financial_sensitive
  - legal_sensitive
  - unclear
- reviewOwner:
  - Rose
  - Carmen
  - Rose and Carmen
  - Assigned team member
  - Unassigned
- status:
  - new
  - needs_review
  - routed
  - approved
  - rejected
  - archived
- duplicateRisk:
  - none
  - possible
  - likely
- relatedProjectIds or relatedProjectNames
- reviewerNotes
- finalActionTaken
- auditLog

4. Classification Logic

Create a classifier function that reviews message text and assigns:
- detectedType
- suggestedDestination
- sensitivity
- reviewOwner
- duplicateRisk
- recommended next step

Initial implementation can be rule-based if no live AI service exists.

Examples:
- Messages containing “Rose and Carmen,” “mind meld,” “Carmenfy,” “Rosify,” “strategy,” or “founder review” should suggest Rose/Carmen Mind Meld.
- Messages containing “Zoho,” “CRM,” “Cliq,” “automation,” “Deluge,” “Flow,” “form,” “field,” or “workflow” should route to Carmen review.
- Messages containing “decision,” “approved,” “final,” “launch,” “pricing,” “proposal,” or “client-facing” should route to Rose review or Rose and Carmen review depending on systems impact.
- Messages containing “todo,” “task,” “follow up,” “remind,” or “need to do” should suggest Command Center Task.
- Messages containing “idea,” “what if,” “we should,” or “new product” should suggest Idea Backlog or CollabOS Review Queue.
- Sensitive messages should not be routed into general Company Brain automatically.

Create a clear review queue with filters:

6. Human Actions

Each intake item should support these actions:

- Send to Rose/Carmen Mind Meld
- Send to CollabOS Review Queue
- Create Draft Task
- Create Draft Idea
- Create Build Request Draft
- Create Company Brain Update Suggestion
- Mark Sensitive / Private
- Mark Duplicate / Link Existing Work
- Archive / No Action

Important:
These actions should create draft or review-ready items only unless the app already has approved persistence and approval workflows.

7. Rose/Carmen Mind Meld Routing

If an item is sent to Rose/Carmen Mind Meld:
- Preserve privacy.
- Do not expose the raw message to non-Rose/Carmen areas.
- Create a safe summary only if needed for broader review.
- Include fields:
  - Rose thoughts
  - Carmen thoughts
  - Rose OS synthesis
  - Open questions
  - Ready to Carmenfy
  - Ready to Rosify
  - Needs both review
  - Sensitive flag

8. Company Brain Update Suggestions

If an item suggests company knowledge:
- Do not write directly to Company Brain.
- Create a proposed update with:
  - proposed record type
  - summary
  - source message
  - reason this should be preserved
  - recommended destination document
  - approval needed from Rose, Carmen, or both

Potential destination documents:
- Active Projects Tracker
- Decision Log
- Build Registry
- Requirements Registry
- Automation Registry
- SOP Library
- CRM Architecture Document
- Meeting Notes Repository
- Knowledge Gap Report

9. Duplicate Work Detection

Add a lightweight duplicate check.

Compare new intake items against existing local/project sample records by:
- project name
- keywords
- system name
- app name
- CRM/form/automation language
- similar idea text

If possible duplicate found:
- Show “Possible duplicate”
- Link related item
- Recommend reviewing before creating new work

10. Privacy And Governance Guardrails

Add visible guardrail copy in the admin/review UI, not as marketing text:
- External messages are intake drafts until reviewed.
- Sensitive/private items must not be shared broadly.
- AI/rule classifications are recommendations, not approved company decisions.
- Rose approves company direction, client-facing content, pricing, launch decisions, and final decisions.
- Carmen reviews systems, process, Zoho, AI, automation, and build architecture.
- High-risk items may require both Rose and Carmen.

13. Design Requirements

The UI should feel like an operational command center, not a marketing page.

Use:
- Clean dashboard layout
- Strong filters
- Scannable queue
- Clear status badges
- Private/sensitive visual treatment
- No yellow-heavy Rose Brain branding
- No fake live claims

14. Acceptance Criteria

The build is complete when:

- External Intake section exists in navigation.
- User can add a manual sample message.
- Sample message is classified into type, destination, sensitivity, reviewer, and status.
- User can route an item to draft task, idea, build request, Company Brain suggestion, or Rose/Carmen Mind Meld.
- Sensitive/private items are visibly protected.
- Integration settings show Zoho Cliq and WhatsApp as off/test/live.
- No UI claims live WhatsApp/Cliq integration unless configured.
- Review queue supports filters and status changes.
- Duplicate-risk warning exists.
- Audit trail records routing/review actions.
- Rose/Carmen Mind Meld receives only private/review-safe items.
- Company Brain updates remain proposed, not automatic.
