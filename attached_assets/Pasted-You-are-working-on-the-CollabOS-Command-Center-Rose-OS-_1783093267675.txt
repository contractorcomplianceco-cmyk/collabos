You are working on the CollabOS Command Center / Rose OS collaboration app.

Expand the app with real backend admin login, role-based access levels, permissions, and a much more innovative Mockup Studio / Idea-to-Build creative engine.

Product names:
- Rose OS = full company operating system and AI assistant experience
- Command Center = work and project dashboard inside Rose OS
- Rose Brain = AI knowledge assistant and memory feature
- Company Brain = approved company records and institutional memory
- CollabOS Command Center = collaboration intelligence layer
- Mind Meld = private Rose/Carmen collaboration and alignment space

Core principle:
This app must feel like a real internal operating system, not a static mockup. It should be intuitive, fun, highly capable, and safe. Do not fake backend login, access control, or live permissions. If something is still demo-only, label it clearly.

SECTION 1: Backend Admin Login

Add a real backend authentication layer.

Required:
- Admin login screen
- Email + password login
- Secure password hashing
- Server-side session or JWT-based authentication
- Logout
- Protected routes
- Current user profile
- Role attached to the logged-in user
- Clear unauthorized / access denied state

Do not store plaintext passwords.
Do not expose backend secrets in frontend code.

If the project already has a backend:
- Extend the existing backend/auth system.
- Do not create a duplicate auth pattern.

If the project does not have a backend:
- Create a simple backend suitable for Replit testing.
- Use a local database or existing project database pattern.
- Keep the code clean and easy to migrate later.

SECTION 2: Access Levels / Roles

Create role-based access control.

Roles:

1. Super Admin
- Full access to all areas.
- Can manage users, roles, settings, integrations, records, and system configuration.

2. Rose / Founder Admin
- Full leadership access.
- Can approve company direction, final decisions, client-facing content, pricing/proposals, public launch decisions, sensitive leadership items, and Company Brain update suggestions.
- Has access to Rose/Carmen Mind Meld.

3. Carmen / Systems Admin
- Full systems/process/build access.
- Can review Zoho, CRM, automations, AI workflows, backend architecture, app builds, integrations, and operational systems.
- Has access to Rose/Carmen Mind Meld.

4. Leadership Reviewer
- Can review assigned items, comment, approve within assigned area, and route items.
- Cannot change system settings or user permissions unless granted.

5. Contributor
- Can submit ideas, tasks, notes, blockers, build suggestions, and intake items.
- Can view their own submitted items and assigned work.
- Cannot approve decisions or Company Brain updates.

6. Viewer
- Read-only access to permitted dashboards and assigned items.
- Cannot submit approvals or change routing.

7. Guest / Demo
- Limited demo access only.
- Cannot view private Mind Meld, sensitive items, integrations, settings, or Company Brain suggestions.

SECTION 3: Permission Matrix

Add a permissions layer so features can check both role and item sensitivity.

Protected areas:
- External Intake
- CollabOS Review Queue
- Rose/Carmen Mind Meld
- Mockup Studio
- Idea Constellation
- Company Brain Suggestions
- Build Prompt Generator
- Integration Settings
- User Management
- System Settings
- Audit Logs

Special privacy rules:
- Rose/Carmen Mind Meld is visible only to Rose, Carmen, and Super Admin unless explicitly configured otherwise.
- Sensitive leadership content is not visible to general contributors.
- HR, legal, financial, client-sensitive, and private leadership items must require elevated permission.
- Company Brain suggestions require approval before becoming official records.
- Integrations and secrets are admin-only.

SECTION 9: Security And Honesty Requirements

- Do not expose passwords or secrets.
- Protect admin routes.
- Protect API routes.
- Do not allow frontend-only role checks for sensitive actions.
- Backend must verify permission before protected action.
- Demo users must be clearly marked.
- If persistence is local/demo only, label it honestly.
- Do not claim production security unless implemented.
- Do not claim live WhatsApp/Cliq/AI/mockup generation unless actually connected.

SECTION 10: Acceptance Criteria

The build is complete when:

- App has a backend-protected login.
- Users have roles.
- Protected sections require permission.
- Admin can manage users.
- Rose/Carmen Mind Meld is private.
- Sensitive items require elevated access.
- Audit logs are created for major actions.
- Mockup Studio exists or is significantly expanded.
- Mockup Studio can create a mockup brief from an idea/intake item.
- User can plan screens for a mockup.
- User can define visual direction.
- User can create/edit mockup versions.
- User can generate a build handoff.
- User can generate a Replit prompt.
- Approval states exist for mockups.
- Dashboard includes high-tech intuitive widgets like Attention Pulse, Approval Passport, or Innovation Lab.
- App does not fake live integrations, AI, or production security.
